Design Rationale: This hybrid pattern keeps primary navigation horizontal at the top (like GitHub / Vercel), giving maximum horizontal space to content. The context sidebar only appears when the current section has meaningful sub-navigation (Cases shows stage filters, Settings shows config categories). Dashboard gets the full viewport width. Secondary nav items (Messages, Email, Contacts, Documents, Reports) live behind a "More" dropdown and are also accessible via ⌘K search. This avoids the left sidebar "wall of links" while still providing quick access to contextual filtering.
